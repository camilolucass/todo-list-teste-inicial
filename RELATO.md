# Relato rápido

Organizei a entrega começando pelos requisitos obrigatórios: cadastro, listagem, edição, exclusão, pesquisa, campos da tarefa e validações. Depois migrei a aplicação para Next.js, React e TypeScript para deixar o projeto mais próximo de uma base profissional de front-end.

Utilizei Next.js, React, TypeScript, CSS, Lucide React e `localStorage` para salvar as tarefas no navegador. Também utilizei IA como apoio no fluxo de desenvolvimento, revisão da implementação e avaliação de integrações de agenda.

Como diferencial, adicionei uma visão de calendário e integração com agenda por link do Google Calendar e exportação `.ics`. Optei por não implementar OAuth direto com Google Calendar API ou Microsoft Graph nesta versão porque isso exigiria credenciais, consentimento do usuário e configuração externa, aumentando a complexidade além do necessário para o teste inicial.
